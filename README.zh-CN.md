# Socrates.SKILL

一个用于 [Claude Code](https://docs.anthropic.com/en/docs/claude-code) 的辩证思维引擎技能，运用**苏格拉底诘问法 + 第一性原理 + 奥卡姆剃刀**进行迭代式深度思考。

这个技能不解决问题。它通过不断追问假设、还原不可约的真理、以最小复杂度重建认知，来深化对问题的理解。

## 工作原理

每一轮迭代执行一个完整的**正题 -> 反题 -> 合题**循环：

1. **正题（Thesis）** -- 用一句清晰的、可证伪的话陈述当前理解
2. **反题（Antithesis）** -- 使用苏格拉底式追问（5种提问类型）挑战正题
3. **合题（Synthesis）** -- 剥离至第一性原理，用奥卡姆剃刀重建

合题成为下一轮的正题。循环持续进行，直到用户中断或**突变守卫**检测到认知边界（连续2轮未产生真正的理解转变）。

## 快速开始

### 安装

将 `skills/socrates/` 目录复制到你项目的技能目录中：

```bash
cp -r skills/socrates/ /path/to/your-project/skills/socrates/
```

或通过 npx 安装：

```bash
npx skills install socrates --from github:MoYeRanqianzhi/Socrates.SKILL
```

### 使用

通过要求 Claude Code 进行深度思考来触发技能：

```
/socrates 我们应该用微服务还是单体架构？
```

或者用自然语言：

```
深入思考我们是否应该接受风投还是自力更生。
质疑我们迁移到 Kubernetes 这个决定背后的假设。
用第一性原理分析民主是否是最好的治理体系。
```

## 技能结构

```
skills/socrates/
├── SKILL.md                          # 核心 SOP（259 行，约 3-4k tokens）
└── reference/
    ├── socratic-questions.md         # 5种提问类型详解（按需加载）
    └── engineering.md                # 工程特化扩展（按需加载）
```

遵循**渐进式披露**原则 -- 只加载需要的内容：

| 层级 | 内容 | 加载时机 | Token 消耗 |
|------|------|----------|-----------|
| L1 | 名称 + 描述 | 始终加载 | ~100 |
| L2 | SKILL.md | 技能被触发时 | ~3-4k |
| L3 | 参考文件 | 按需引用 | 无限制 |

## 五种苏格拉底提问类型

| 类型 | 使用场景 |
|------|---------|
| **澄清（Clarification）** | 术语模糊、定义被默认、范围不清 |
| **假设追问（Assumption Probing）** | 正题依赖未经检验的前提 |
| **证据追问（Evidence Probing）** | 主张缺乏论证或证据薄弱 |
| **视角转换（Perspective Shifting）** | 仅考虑了单一视角 |
| **后果推演（Consequence Tracing）** | 推论未被充分探索，可能导致矛盾 |

## 双模式

### 独立模式（面向用户）

输出可见的思维链，包含结构化的轮次：

```
## 第 N 轮

### 正题
> 当前理解

### 反题
**提问类型**：假设追问
[追问过程...]
**被动摇的假设**：[被质疑的内容]

### 合题
**不可约的事实**：[...]
**最简重建**：[新的理解]
**突变**：是 -- [发生了什么转变]
```

### 引擎模式（被其他技能调用）

静默运行，仅返回最终合题：

```
[socrates-config]
max_rounds: 3
focus: "架构决策"
return: "final_synthesis"
[/socrates-config]
```

## 核心机制

### 突变守卫（Mutation Guard）

防止循环退化为伪装成进步的重复。每轮合题之后：
- **检测到突变** -> 进入下一轮
- **未检测到突变** -> 切换提问类型，尝试更深层假设
- **连续2轮失败** -> 宣告认知边界（由用户决定下一步）

换个说法不是突变。重新排序不是突变。添加限定词不是突变。只有真正的认知转变才算。

### 工程特化扩展

当问题涉及软件工程时自动激活。增加：
- 工程领域专用的追问维度
- 工程第一性原理（物理约束、理论极限、资源约束）
- 常见假设陷阱（微服务扩展性、缓存性能、NoSQL 处理非结构化数据等）

## 硬性规则

1. 永远不跳过反题
2. 永远不把重复伪装成突变
3. 合题中永远不诉诸权威
4. 永远不过早收敛
5. 每一轮必须暴露至少一个假设

## 评估结果

采用 TDD 方法测试（RED -> GREEN）：

- **基线（无技能）**：0 次自我追问、先给结论、以权威为证
- **使用技能后**：24/24 断言通过，所有基线缺陷已解决
- **引擎模式**：正确静默运行，遵守 max_rounds 限制
- **突变守卫**：正确检测到收敛，宣告认知边界
